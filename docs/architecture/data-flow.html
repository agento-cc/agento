Tool calls

cron ⇄ toolbox · HTTP (no shared filesystem, no shared secrets)
agent CLI → mcp__toolbox__* → HTTP /sse or /mcp → toolbox MCP server
cron
Agent invokes MCP tools
Tool calls flow over the MCP connection defined in .mcp.json. The agent has no credentials of its own — every privileged action is delegated to the toolbox.
Each call is recorded in the agent transcript (~/.claude/projects/*/<session_id>.jsonl or ~/.codex/sessions/.../rollout-*.jsonl). Stage 6's verification gate parses this file.

Finalize — stats, verdict, status commit

cron container · _finalize_job
cron
Collect _JobResult
consumer.py:55 (struct), consumer.py:575 _finalize_job — summary, agent_type, model, input/output tokens, prompt, output, session_id.
↓
MCP-health telemetry (rc=0 path)
consumer.py:637 dispatches with JobFinalizeEvent(job, job_result, elapsed_ms, provider, verdict=None). Observer McpHealthTelemetryObserver (app_monitor/events.json, order=100) parses the transcript via get_transcript_reader(provider) and records two nullable signals on the job row — toolbox_mcp_calls (count of mcp__toolbox__* tool-uses) and toolbox_mcp_connected (from job_result.mcp_init) — then optionally emails ops. It is telemetry only: verdict stays None, so an rc=0 job always commits SUCCESS. The framework's verdict plumbing remains for future modules, but no in-tree observer sets a verdict.

Alerts

cron container · app_monitor module
cron
AlertEmailObserver on job_dead_after
Wired via app_monitor/events.json (order=100). Composes subject + body — job id, reference_id, source, attempt count, error class + message; if the failure was a JobVerificationFailed, includes verdict.reason and verdict.detail. Sends via stdlib SMTP using config keys alerts/email_to, alerts/smtp_host, etc.
job_dead_after
toolbox
Not involved — SMTP credentials would live in the toolbox if any tool needed them, but the alert path uses the host's stdlib SMTP from the cron container.

Notes on the container split

  • Agent ⇄ toolbox communication is HTTP only (http://toolbox:3001); the agent never holds credentials.
  • Stages 1–4, 6, 7 happen entirely in the cron container. Stage 5 is the only point where the toolbox container is active.
  • Verification (Stage 6) reads transcripts on the cron container's filesystem — it does not call the toolbox.
