# v0.7.0 Migration — Agent Identity Moves Into the Database

**This is a hard break.** There is no backward-compat path: after upgrading, any workspace that relied on filesystem-mounted SSH keys or JSON credential files must re-register them via the new CLI. Existing OAuth token rows will load as "credentials missing" until re-registered.

## What Changed

| Before (≤ 0.6.x) | After (0.7.0) |
|---|---|
| `../../id_rsa` mounted read-only into `sandbox` and `cron` containers at `/run/secrets/id_rsa`, then copied to `/home/agent/.ssh/id_rsa` by each entrypoint | Per-agent_view SSH key stored encrypted in `core_config_data` under `agent_view/identity/ssh_private_key`. `workspace:build` materializes it into `<build_dir>/.ssh/id_rsa` (mode 0600). Consumer sets `HOME=<build_dir>` when spawning the agent subprocess. |
| OAuth credentials JSON files under `/etc/tokens/<label>.json` (`tokens/` bind-mounted into containers). `oauth_token.credentials_path` pointed at the file. | OAuth credentials stored encrypted inline in `oauth_token.credentials` (TEXT, AES-256-CBC). The host `tokens/` directory is no longer mounted. |
| `$HOME = /home/agent`, with symlinks `~/.claude → /workspace/.claude`, `~/.codex → /workspace/.codex` wired by `entrypoint.sh`. All agents on the host shared this HOME. | `$HOME = workspace/build/<ws>/<av>/builds/<id>/` — per-agent_view, populated by `workspace:build`. Session history lives in a sibling `state/` directory, symlinked into HOME. |
| `workspace_build` did not retain multiple builds on disk beyond the previous one. | Up to `workspace_build/retention/max_builds` (default 10) previous build directories are retained. Older ones are garbage-collected. |

## Required Actions After Upgrade

1. **Pull the new image / upgrade the CLI.**
2. **Apply schema migration:**
   ```bash
   agento setup:upgrade
   ```
   This runs `019_oauth_token_inline_credentials.sql` (drops `credentials_path`, adds `credentials`).
3. **Re-register every active OAuth token** — the new column starts empty, so the DB row exists but is unusable until re-registered:
   ```bash
   agento token:register claude prod-1 /path/to/claude_oauth_1.json
   agento token:set claude <id>
   ```
   You can safely delete the source JSON file afterwards.
4. **Register SSH keys per agent_view** via the generic `config:set` (paste interactively or pipe — the field is `type: "obscure"`, so encryption is automatic). Use `--scope default` / `--scope workspace --scope-id <id>` if you truly need a single global key:
   ```bash
   cat ~/.ssh/agent_dev_01 | agento config:set agent_view/identity/ssh_private_key --agent-view dev_01
   cat ~/.ssh/agent_dev_01.pub | agento config:set agent_view/identity/ssh_public_key --agent-view dev_01
   ```
5. **Rebuild all workspaces** so identity material is materialized into the new build structure:
   ```bash
   agento workspace:build --all
   ```
6. **(Optional) Remove now-unused on-host artifacts:**
   ```bash
   rm /path/to/agento/../id_rsa
   rm -rf /path/to/agento/tokens/
   ```
   Both the `tokens/` bind-mount and the `id_rsa` read-only secret mount have been removed from `docker/docker-compose.dev.yml` and the customer `docker-compose.yml` template.

## For Custom Agent Modules

The `ConfigWriter` protocol gained one new optional method — `persistent_home_paths() -> list[str]`. Return relative-to-HOME paths that must survive rebuilds (session history, tool caches, anything the agent writes stateful data into). Example from the Claude module:

```python
def persistent_home_paths(self) -> list[str]:
    return [".claude/projects", ".claude/todos"]
```

`workspace:build` aggregates these across all registered ConfigWriters and symlinks each into the per-agent_view `state/` directory. No changes to `workspace_build` are required when adding a new agent module — register your `ConfigWriter` via `di.json` as before.

## Verification

```bash
# Schema applied
agento setup:upgrade --dry-run | grep 019

# Token re-registered
agento token:list

# Identity materialized
agento agent_view:identity:show dev_01
agento workspace:build --agent-view dev_01
ls -la workspace/build/it/dev_01/builds/*/.ssh/id_rsa    # mode 0600
readlink workspace/build/it/dev_01/builds/*/.claude/projects   # → ../../../state/.claude/projects

# Retention
agento config:get workspace_build/retention/max_builds
```

## Rollback

Not supported. Downgrading past 0.7.0 requires restoring your database from a pre-upgrade snapshot and putting back the deleted `id_rsa` / `tokens/` files.
